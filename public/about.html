Objetivos

  • Propor uma nova forma de representação visual das requisições a um servido web
  • Comunicar a informação de maneira clara e efetiva utilizando meios gráficos
  • Processar uma quantidade muito grande de dados e extrair as informações relevantes
  • Representar informações que outrora, devido a sua imensa quantidade, eram de dificil análise

Etapas do processo de extração e visualização

  • Redirecionamento do log do servidor web para uma máquina ubuntu linux hospedada no EC2 Amazon
  • Utilização do Syslog no EC2 para filtar apenas o log vindo do Eventick
  • Utilização do crontab do linux para enviar o log diário para um repositório de dados no S3 da Amazon
  • Contrução de um Webservice em Ruby on Rails hospedado no Heroku e responsável por:
    • Parsear o log e extrair as informações necessárias
    • Persistir os dados em um banco Postgres
    • Construir uma pequena API que responde a requisições em formato JSON
  • Construção de uma aplicação em Sinatra.rb para servir páginas html
  • Utilização de Javascript para plotar os gráficos
